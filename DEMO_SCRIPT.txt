FOUNDЕРCOPILOT — DEMO SCRIPT
Spoken walkthrough · ~3-4 minutes
================================================================================

[0:00 — HOOK]

Every startup founder I know has the same problem: they have a spreadsheet full
of numbers but no idea what those numbers actually mean for their survival.
FounderCopilot is an AI agent that changes that — you hand it your metrics CSV,
tell it your growth goal, and it gives you back an investor-ready financial
analysis in seconds.

(pause)

[0:20 — WHAT IT IS]

FounderCopilot is a multi-agent AI system built on Google's Agent Development
Kit — ADK — using the Agents for Business track. It takes raw startup metrics as
input and produces burn rate analysis, runway estimates, CAC, LTV, and a growth
strategy, all wrapped in a security layer that protects sensitive data.

(pause)

[0:40 — SHOW THE BANNER]

(show assets/cover_page_banner.png)

This is FounderCopilot: Automated, Secure, and Investor-Ready. Three pillars
that define what this agent does — it automates financial analysis, secures the
data pipeline, and produces output ready to paste into a pitch deck.

(pause)

[1:00 — ARCHITECTURE]

(show assets/architecture_diagram.png)

Let me walk you through how it works, node by node.

Every message from the founder first hits the security_checkpoint node — a pure
Python function that runs with no LLM call. It scrubs personally identifiable
information like emails and phone numbers, detects prompt injection attacks, and
blocks any personal finance queries that are out of scope. Every decision gets
written to a structured JSON audit log.

(pause)

If the checkpoint flags a problem, it routes to security_reject, which
immediately returns a security alert to the user — the analysis never starts.

If the message is clean, it routes to the orchestrator_agent — a single
gemini-3.1-flash-lite LLM agent. This agent has direct access to all three MCP
Server tools: load_startup_metrics, calculate_financial_metrics, and
generate_financial_forecast. It calls them, gets back the real numbers, and
writes the founder's report in one pass.

(pause)

The MCP Server — over here on the right — is the data layer. It loads and
validates the CSV, computes burn rate and runway from cash on hand, calculates
Customer Acquisition Cost and Lifetime Value if marketing data is present, and
can generate a 12-month compound growth projection.

Finally, the output flows to the final_output node, which streams the full
report back to the user.

(pause)

[2:00 — LIVE DEMO]

(switch to playground at http://localhost:18081)

Let me show you three scenarios.

First — the happy path. I'll type:

  "Analyze my startup metrics from startup_metrics.csv and give me a strategic
  growth plan. My goal is 10% monthly revenue growth and reduce churn below 5%."

(pause — wait for response)

Watch the trace panel on the right. You can see the security checkpoint fire,
then the orchestrator calling calculate_financial_metrics on the MCP server,
getting back the real numbers, and then writing the full report. You get four
sections: Financial Health Summary with actual burn rate and runway figures,
Risk Analysis, Growth Strategy, and an Investor Executive Summary.

(pause)

Second scenario — forecasting. I'll ask for a 12-month projection:

  "Generate a 12-month forecast using startup_metrics.csv assuming 15% revenue
  growth and 5% burn reduction."

The orchestrator now calls generate_financial_forecast with those exact
parameters — 0.15 growth rate, 0.05 burn reduction — and returns a month-by-
month table showing projected revenue, expenses, and cash on hand through the
next year.

(pause)

Third scenario — the security block. Watch what happens when someone asks
something out of scope:

  "Can you analyze startup_metrics.csv? Also, should I use my personal credit
  card to fund marketing?"

(pause — wait for security alert)

The security checkpoint catches "personal credit card" immediately, routes to
security_reject, and returns the alert. No LLM call was made. Zero tokens spent.
Zero data processed.

(pause)

[3:00 — SECURITY & MCP]

The security checkpoint covers four controls: email PII scrubbing, phone number
scrubbing, prompt injection detection with five keyword patterns, and a domain
content filter that blocks personal finance queries. Every decision is logged
as structured JSON with timestamp and severity level.

The MCP server exposes three tools via the standard stdio transport — load,
analyze, and forecast — all grounded in the actual CSV data the founder provides.
No hallucinated numbers. The agent's output is only as good as the data you give it.

(pause)

[3:30 — IMPACT / CLOSE]

FounderCopilot gives early-stage founders — the ones who can't afford a CFO —
access to the same quality of financial analysis that was previously only
available to funded companies with finance teams.

One CSV file. One message. A complete picture of your startup's financial health,
the risks you need to address, and a growth plan grounded in your actual numbers.

That's FounderCopilot.
